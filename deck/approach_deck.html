SignalRank AI

Explainable Candidate Discovery Engine

Redrob / Hack2skill Data & AI Challenge

Problem

Recruiting search fails when it ranks keywords instead of real fit.

SignalRank AI separates shipped career evidence from shallow AI buzzwords.

What the JD Really Needs

  • Production ranking, search, retrieval, and recommendations
  • Embeddings and vector search
  • Evaluation frameworks and product engineering
  • Availability and hiring readiness

System Architecture

Candidate data -> parsing -> features -> static fit
  -> Redrob behavior -> trap penalties -> ranking
  -> deterministic reasoning -> CSV/XLSX

Static Fit Scoring

  • Career evidence and retrieval/ranking proof
  • Skills as support, not proof
  • Experience, product context, location, and evaluation

Redrob Behavior Scoring

  • Activity, response rate, notice, relocation, reliability
  • Profile trust and technical activity
  • Narrow multiplier so behavior does not overpower fit

Trap Defense

  • Wrong-role keyword stuffing
  • Weak AI hype without production evidence
  • Impossible or inconsistent profile signals
  • Non-target AI-only profiles

Reasoning

  • Specific, factual, JD-connected, and concern-aware
  • Generated from extracted evidence and scorecards
  • No hosted LLM calls during ranking

Runtime and Reproducibility

  • Full 100K CSV-only run: about 3m16s-3m23s locally
  • CPU-only and no network during ranking
  • Deterministic repeat hash

Demo and Submission

  • Public GitHub repo and Streamlit sandbox
  • Canonical CSV for validator
  • Synced XLSX for portal convenience
  • PDF approach deck
